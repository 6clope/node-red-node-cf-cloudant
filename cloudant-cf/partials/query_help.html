A node for searching documents in a Cloudant database.

Searching for documents can be done in three modes: directly by using the document's _id, by using an existing Search Index or retrieving all documents stored in your database.

When querying using the _id option, the value for the document's _id should be passed in the msg.payload as a string.

To use an existing Search Index stored on the desired database, the query argument should be passed on the msg.payload as a string following the indexName:value pattern. Keep in mind that the index must be created beforehand in the database. and referenced here by its design document/index name.

When querying using a Search Index you can pass the search parameters as an object in msg.payload.

For example, you can pass an object like this:

{ query: "abc*", limit: 100 }

to change the value of limit. You can find more information about Search Index parameters in the official Cloudant documentation.

The last method to retrieve documents is to simply get all of them by selecting the option all documents.

The database name must follow these rules:

  • No spaces
  • All letters in small caps
  • The first character can't be _
