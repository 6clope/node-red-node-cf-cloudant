A simple Cloudant output node. Stores msg in a chosen database.

Cloudant generates a random id for the _id property if one is not specified and a new document will be inserted each time. You can insert only the msg.payload by checking the checkbox in the configuration window.

If you want to update an existing document you must specify the latest values for _id and _rev in your object. If they don't match the current values stored in the database you will receive a conflict error message, so make sure you are using the most updated information for the document.

It is also possible to delete documents from the database by providing values for _id and _rev and selecting the remove option for the node. You can pass these values in the msg object itself or as an object in the msg.payload.

The database name must follow these rules:

  • No spaces
  • All letters in small caps
  • The first character can't be _

Your document should avoid having top-level fields that start with _, with exceptions for _id, _rev and other CouchDB reserved words.
